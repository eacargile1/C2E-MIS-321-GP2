## Deep story analysis (patterns)

### Common struggles / complexity

- **Bootstrap + security hardening (S-E1-01):** Early reviews surfaced JWT expiry vs config drift, empty CORS origins crashing startup, and secrets in committed config — predictable for a greenfield API and addressed with patches and env/user-secrets guidance.
- **Session vs account state (S-E1-02):** Deactivated users retaining access until token expiry led to **RequireActiveUserMiddleware** and JWT `active` claim — a real lesson that “login” alone is not enough for account lifecycle stories.
- **Story overlap (S-E1-03):** Much of FR3 was implemented with S-E1-02; the story correctly framed **verification + gap-fill** rather than duplicate work. Remaining gaps were test tightening (case-insensitive role patch, promote IC→Admin when a second admin exists).

### Recurring review themes

- **Consistency:** Align HTTP semantics (401 anonymous vs 403 wrong-role on RBAC stubs), shared **`AuthMessages.Forbidden`**, and JSON error shape across `UsersController` and new stubs.
- **Hardening deferred explicitly:** Rate limiting on login, real persistence (vs in-memory EF), JWT idle timeout — tracked, not silently forgotten.

### Technical debt captured in stories

| Area | Severity | Notes |
|------|----------|--------|
| In-memory database | High for production | Deployable persistence deferred; called out in S-E1-01+ |
| Login rate limiting | Medium | Deferred MVP throttle |
| `GET /api/auth/me` bare 401 vs JSON in some paths | Low | Documented inconsistency |
| Last-admin protection TOCTOU | Low (MVP) | Concurrency-safe pattern deferred |
| `AdminAndFinance` coupling in `RbacRoleSets` | Low | Split when matrix rows diverge |
| `web/dist/` tracked | Repo hygiene | Consider gitignore + CI build |

### Testing / quality

- Integration tests scaled from auth-only through **UsersAdminTests** to **RbacEnforcementTests**; unique in-memory DB names per factory avoided parallel pollution.
- Test count growth noted in story records (single-digit → mid-20s by E1 end) — good signal for regression safety on Epic 2.
